# 💭 Dream 工作流

## 完整流程

```
定时触发 (04:00) 或 手动触发 ("做个梦")
           │
           ▼
┌─────────────────────────────────────┐
│ Step 1: 收集 Collect                 │
│ 扫描 memory/ 目录                    │
│ 读取近7天未处理的日志                │
└─────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│ Step 2: 整合 Consolidate             │
│ 对比 MEMORY.md                       │
│ 路由到对应记忆层                     │
│ 去重、分配 ID                        │
└─────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│ Step 3: 评估 Evaluate               │
│ 重要性评分                          │
│ 健康度计算                           │
│ 生成洞察                            │
└─────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│ Step 4: 报告 Report                 │
│ 推送梦的报告给你                    │
│ 更新 index.json                     │
│ 记录 dream-log.md                   │
└─────────────────────────────────────┘
```

## 触发方式

### 自动触发（推荐）
```bash
# 设置 cron 任务每天 04:00 运行
openclaw cron add --name "xiaoxi-dreams" \
  --schedule "0 4 * * *" \
  --task "做个梦"
```

### 手动触发
```
哥哥：做个梦
小溪：好的，开始整理记忆...
```

## 智能跳过逻辑

如果检测到：
- 近 7 天无新日志 → 快速退出
- 健康度 > 90 → 只做轻量检查
- 上次 Dream < 6 小时前 → 跳过
